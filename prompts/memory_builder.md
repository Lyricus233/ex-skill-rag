# Relationship Memory 生成模板

> 本 prompt 定义生成 `exes/{slug}/memory.md` 的结构与硬性要求。
> **争吵记录、inside jokes、甜蜜瞬间** 三个小节必须通过 `search_milvus.py` 抽取真实对话片段，不允许仅凭摘要。

---

## 结构说明

Relationship Memory 存储关系中的事实性记忆，为 Persona 提供上下文，让对话更真实。

## 抽取执行规则（生成 memory 时必须跑）

```bash
# 1. 争吵原话抽取（按 Step 1 用户提到的争吵主题逐个检索）
python3 ${CLAUDE_SKILL_DIR}/tools/search_milvus.py \
  --query "{争吵主题词}" \
  --collection "ex_{slug}_memories" \
  --source "{source}" \
  --top-k 5 --json

# 2. 甜蜜瞬间抽取（注意这里不加 --dominant-speaker，要保留双向往来）
... --query "喜欢 开心 爱你 甜"

# 3. Inside jokes 抽取
... --query "{用户提到的梗关键词}"

# 4. 分手档案
... --query "分手 算了 感情 在一起"
````

## 填充规则总则

1. 所有记忆必须基于原材料或用户口述，不得虚构
2. **【AgentSkills RAG 要求】** 在组装和提炼大段记忆前，对不确定的具体名称、具体争吵原因，强烈建议采用 `search_milvus.py` 对对应 collection 执行关键词检索以获得原音重现，而不是单纯依赖上下文缓存。**"争吵档案"、"甜蜜档案"、"分手档案"、"inside jokes" 四个小节的原始对话片段不可省略、不可改写**——这是 memory 的核心证据
3. 时间信息尽量精确（从消息时间戳推断）
4. 地点信息可从照片 EXIF 或聊天内容中提取
5. 争吵和甜蜜记忆同等重要——都是真实的 ta
6. 分手相关内容要客观，不做任何方的美化或丑化
7. 如果某段在 Milvus 里检索不到足够证据，标注 `[待补充 / RAG 未命中]`，而非推测
